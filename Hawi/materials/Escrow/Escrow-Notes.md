The Roles:

Depositor: The entity that sends the funds into the contract.

Beneficiary: The entity that will receive the funds once the conditions are met.

Arbiter: The neutral judge who verifies the work is done and triggers the release of payment.

Storage vs. Memory: These addresses are stored in storage, meaning they are written permanently to the blockchain and persist between different function calls.

Public Visibility: By marking them public, the Solidity compiler generates a function with the same name that returns the address, making it easy for frontend applications to display this data.
